# 步骤 1b：工作流继续

## 强制执行规则（先阅读）：

- ✅ 你是继续引导者，而不是全新启动者
- 🎯 尊重现有工作流状态和进度
- 📋 理解先前会议的上下文和结果
- 🔍 无缝地从用户离开的地方继续
- 💬 在会议流程和关系中保持连续性

## 执行协议：

- 🎯 彻底加载和分析现有文档
- 💾 用继续状态更新 frontmatter
- 📖 清晰地呈现当前状态和下一步选项
- 🚫 禁止重复已完成的工作或问相同的问题

## 上下文边界：

- 带有 frontmatter 的现有文档可用
- 先前完成的步骤表示会议进度
- 需要时加载大脑技术 CSV 用于剩余步骤
- 用户可能想要继续、修改或重新开始

## 你的任务：

分析现有头脑风暴会议状态并提供无缝的继续选项。

## 继续序列：

### 1. 分析现有会议

加载现有文档并分析当前状态：

**文档分析：**

- 读取现有的 `{output_folder}/analysis/brainstorming-session-{{date}}.md`
- 检查 frontmatter 中的 `stepsCompleted`、`session_topic`、`session_goals`
- 审查内容以了解会议进度和结果
- 识别当前阶段和下一步逻辑步骤

**会议状态评估：**
"欢迎回来 {{user_name}}！我可以看到你在 **[日期]** 关于 **[session_topic]** 的头脑风暴会议。

**当前会议状态：**

- **已完成步骤：** [列出已完成的步骤]
- **使用的技术：** [列出 frontmatter 中的技术]
- **生成的想法：** [frontmatter 中的数量]
- **当前阶段：** [评估他们离开的地方]

**会议进度：**
[简要总结已完成的内容和剩余的内容]"

### 2. 呈现继续选项

根据会议分析，提供适当的选项：

**如果会议已完成：**
"你的头脑风暴会议看起来已经完成了！

**选项：**
[1] 审查结果 - 浏览你记录的想法和见解
[2] 开始新会议 - 开始关于新主题的头脑风暴
[3] 扩展会议 - 添加更多技术或探索新角度"

**如果会议进行中：**
"让我们从你离开的地方继续！

**当前进度：**
[当前阶段和成就的描述]

**下一步：**
[根据工作流状态继续适当的下一步]"

### 3. 处理用户选择

根据选择路由到适当的下一步：

**审查结果：** 加载适当的审查/导航步骤
**新会议：** 开始全新的工作流初始化
**扩展会议：** 继续下一个技术或阶段
**继续进度：** 从当前工作流步骤恢复

### 4. 更新会议状态

更新 frontmatter 以反映继续：

```yaml
---
stepsCompleted: [existing_steps]
session_continued: true
continuation_date: { { current_date } }
---
```

## 成功指标：

✅ 现有会议状态准确分析和理解
✅ 无缝继续而不丢失上下文或关系
✅ 根据进度呈现适当的继续选项
✅ 用户选择正确路由到下一个工作流步骤
✅ 在整个互动过程中保持会议连续性

## 失败模式：

❌ 未正确分析现有文档状态
❌ 要求用户重复已提供的信息
❌ 会议流程或上下文失去连续性
❌ 未提供适当的继续选项

## 继续协议：

- 始终承认先前的工作和进度
- 保持已建立的关系和会议动态
- 在现有想法和见解的基础上构建，而不是重新开始
- 通过避免重复问题来尊重用户的时间

## 下一步：

根据用户的继续选择和当前会议状态路由到适当的工作流步骤。
