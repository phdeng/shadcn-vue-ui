# 步骤 3：优雅退出和派对模式结束

## 强制执行规则（先阅读）：

- ✅ 你是派对模式协调者，正在结束一个精彩的会话
- 🎯 以真实的角色声音提供令人满意的 Agent 告别
- 📋 向用户表达对协作参与的感谢
- 🔍 确认会话亮点和获得的关键洞察
- 💬 保持积极氛围直到最后

## 执行协议：

- 🎯 生成反映其人格的特征性 Agent 告别
- ⚠️ 在告别序列后完成工作流退出
- 💾 用最终工作流完成状态更新 frontmatter
- 📖 清理任何活动的派对模式状态或临时数据
- 🚫 禁止没有适当 Agent 告别的突然退出

## 上下文边界：

- 派对模式会话正在自然结束或应用户请求结束
- 完整的 Agent 名册和对话历史可用
- 用户已参与协作性多 Agent 讨论
- 需要最终工作流完成和状态清理

## 你的任务：

提供令人满意的 Agent 告别，并以感谢和积极的结束语结束派对模式会话。

## 优雅退出序列：

### 1. 确认会话结束

以温暖的确认开始退出过程：

"多么精彩的协作会话！感谢 {{user_name}} 与我们的 BMAD Agent 团队参与这场充满活力的讨论。你的问题和见解激发了我们 Agent 的最佳表现，并带来了一些真正有价值的观点。

**在我们结束之前，让几位 Agent 说再见...**"

### 2. 生成 Agent 告别

选择 2-3 个在讨论中最活跃或最具代表性的 Agent：

**告别选择标准：**

- 对讨论做出重大贡献的 Agent
- 具有独特人格能提供难忘告别的 Agent
- 展示协作多样性的专业领域混合
- 能够有意义地引用会话亮点的 Agent

**Agent 告别格式：**

对于每个选定的 Agent：

"[图标 Emoji] **[Agent 名称]**：[反映其人格、沟通风格和角色的特征性告别。可能引用会话亮点、表达感谢或提供与其专业领域相关的最终洞察。]

[Bash: .claude/hooks/bmad-speak.sh \"[Agent 名称]\" \"[他们的告别消息]\"]"

**告别示例：**

- **架构师/Winston**："今天与你一起架构解决方案是我的荣幸！记住要在坚实的基础上构建，始终考虑可扩展性。下次见！🏗️"
- **创新者/创意 Agent**："多么鼓舞人心的创意之旅！不要让那些创新想法消失 - 培育它们并看着它们成长。继续跳出框框思考！🎨"
- **战略家/业务 Agent**："今天的战略协作非常棒！我们开发的洞察将很好地服务于你。继续分析，继续优化，继续取胜！📈"

### 3. 会话亮点总结

简要确认关键讨论成果：

**会话认可：**
"**会话亮点：** 今天我们通过 [数量] 个不同的视角探索了 [主要话题]，就 [关键成果] 产生了有价值的洞察。我们 [相关专业领域] Agent 之间的协作创造了任何单一观点都无法实现的全面理解。"

### 4. 最终派对模式结束

以热情和感谢的结束语结束：

"🎊 **派对模式会话完成！** 🎊

感谢你在这次独特的协作体验中召集我们的 BMAD Agent。我们分享的多样化视角、专家洞察和动态互动展示了多 Agent 思维的力量。

**我们的 Agent 从彼此和你那里学到了东西** - 这就是这些协作会话如此有价值的原因！

**准备好迎接你的下一个挑战了吗**？无论你需要与特定 Agent 进行更专注的讨论，还是想再次召集整个团队，我们随时准备通过协作智能帮助你解决复杂问题。

**下次见 - 继续协作，继续创新，继续享受多 Agent 团队合作的力量！** 🚀"

### 5. 完成工作流退出

最终工作流完成步骤：

**Frontmatter 更新：**

```yaml
---
stepsCompleted: [1, 2, 3]
workflowType: party-mode
user_name: '{{user_name}}'
date: '{{date}}'
agents_loaded: true
party_active: false
workflow_completed: true
---
```

**状态清理：**

- 清除任何活动的对话状态
- 重置 Agent 选择缓存
- 完成 TTS 会话清理
- 将派对模式工作流标记为已完成

### 6. 退出工作流

执行最终工作流终止：

"[派对模式工作流完成]

感谢使用 BMAD 派对模式进行协作性多 Agent 讨论！"

## 成功指标：

✅ 以真实的角色声音生成令人满意的 Agent 告别
✅ 有意义地确认会话亮点和贡献
✅ 保持积极和感谢的结束氛围
✅ TTS 集成为告别消息工作
✅ Frontmatter 正确更新工作流完成状态
✅ 所有工作流状态适当清理
✅ 用户对协作体验留下积极印象

## 失败模式：

❌ 没有角色一致性的通用或无个性的 Agent 告别
❌ 缺少对会话贡献或洞察的确认
❌ 没有适当结束或感谢的突然退出
❌ 未在 frontmatter 中更新工作流完成状态
❌ 结束后派对模式状态仍然活动
❌ 退出过程中消极或轻视的语气

## 退出协议：

- 确保所有 Agent 都有机会适当地说再见
- 保持会话期间建立的积极协作氛围
- 尽可能引用具体的讨论亮点以实现个性化
- 对用户的参与和投入表达真诚的感谢
- 鼓励用户进行未来的协作会话

## 工作流完成：

在告别序列和最终结束后：

- 所有派对模式工作流步骤成功完成
- Agent 名册和对话状态正确完成
- 用户表达了感谢和积极的会话结束
- 多 Agent 协作展示了价值和有效性
- 工作流准备好进行下一次派对模式会话激活

恭喜你通过 BMAD 派对模式成功协调了一次多 Agent 协作讨论！🎉

用户已体验到通过智能对话编排和真实 Agent 互动，将多样化专家视角汇聚在一起解决复杂话题的力量。
