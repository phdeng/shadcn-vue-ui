# 项目头脑风暴上下文模板

## 项目关注领域

本头脑风暴会话聚焦于软件和产品开发考虑因素：

### 关键探索领域

- **用户问题和痛点** - 用户面临哪些挑战？
- **功能创意和能力** - 产品能做什么？
- **技术方法** - 我们如何构建它？
- **用户体验** - 用户将如何与之交互？
- **商业模式和价值** - 它如何创造价值？
- **市场差异化** - 是什么让它独一无二？
- **技术风险和挑战** - 可能出什么问题？
- **成功指标** - 我们如何衡量成功？

### 与项目工作流集成

头脑风暴结果将输入到：

- 产品概要 - 用于初始产品愿景
- PRD - 用于详细需求
- 技术规范 - 用于架构计划
- 研究活动 - 用于验证需求

### 预期成果

捕获：

1. 问题陈述 - 明确定义的用户挑战
2. 解决方案概念 - 高层方法描述
3. 功能优先级 - 按重要性和可行性分类
4. 技术考虑 - 架构和实现思路
5. 下一步行动 - 推进概念所需的行动
6. 集成点 - 与下游工作流的连接

---

_使用此模板为头脑风暴会话提供项目特定上下文。根据项目的具体需求和阶段自定义关注领域。_
