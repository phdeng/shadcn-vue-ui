# 代码审查 - 高级开发人员审查验证清单

## 审查前检查

### 故事状态

- [ ] 故事在冲刺状态文件中标记为"进行中"
- [ ] 故事 ID 与代码审查工作流中使用的 ID 一致
- [ ] 没有其他开发人员正在审查同一故事

### 故事背景文档

- [ ] 故事文档存在于指定位置
- [ ] 故事包含明确的接受标准
- [ ] 故事上下文文档已完成
- [ ] 技术考虑已记录

### 接受标准验证

- [ ] 所有接受标准都有对应的代码更改
- [ ] 没有遗漏的需求
- [ ] 测试覆盖了所有接受标准

## 代码质量审查

### 代码结构

- [ ] 代码遵循项目架构
- [ ] 代码组织清晰且易于跟踪
- [ ] 命名约定一致
- [ ] 没有不必要的复杂性

### 测试覆盖

- [ ] 单元测试覆盖新的/修改的代码
- [ ] 集成测试验证端到端流程
- [ ] 测试清晰且易于理解
- [ ] 没有测试被跳过或禁用

### 文件清单验证

- [ ] 所有修改的文件都已列出
- [ ] 没有意外的文件更改
- [ ] 删除的文件是故意的
- [ ] 没有生成的文件被提交

## 安全审查

- [ ] 没有硬编码的凭据或密钥
- [ ] 没有 SQL 注入漏洞
- [ ] 没有 XSS 漏洞
- [ ] 输入验证到位
- [ ] 身份验证/授权逻辑正确
- [ ] 不敏感数据没有被记录

## 最终签署

- [ ] 通过所有检查
- [ ] 反馈已解决
- [ ] 代码已获批
- [ ] 准备合并到主分支
