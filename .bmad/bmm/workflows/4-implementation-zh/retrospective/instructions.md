# 回顾工作流 - 史诗完成评审

## 工作流概述

完成史诗时，运行此工作流进行团队回顾并庆祝完成。此工作流使用"聚会模式"进行友好的、对话式的审查。

## 核心目标

1. **庆祝完成** - 认可团队成就
2. **学习转移** - 捕获要应用到后续史诗的经验
3. **改进识别** - 确定流程中的改进区域
4. **知识保留** - 记录所有关键发现和决策

## 工作流阶段

### 阶段 1：准备

- [ ] 史诗中的所有故事都标记为"完成"
- [ ] 所有代码已审查和合并
- [ ] 没有未解决的问题或技术债
- [ ] 回顾参与者已确认

### 阶段 2：聚会模式对话

**聚会模式格式**：对话式问答，轻松友好的语气

**讨论主题**:

1. **亮点**：此史诗中进展最顺利的是什么？
2. **挑战**：我们遇到的最大挑战是什么，我们如何克服的？
3. **学习**：我们学到了什么应该应用到下一个史诗？
4. **改进**：如果我们重新做，会有什么不同？
5. **质量**：代码质量和测试覆盖是否令人满意？
6. **沟通**：团队间的沟通有多有效？

### 阶段 3：问题记录

为此回顾创建完成备注记录：

```markdown
## 史诗 {{号}} 回顾 - {{日期}}

### 亮点

- {{亮点 1}}
- {{亮点 2}}

### 挑战和解决方案

- {{挑战 1}} → {{解决方案}}
- {{挑战 2}} → {{解决方案}}

### 关键学习

- {{学习 1}}
- {{学习 2}}

### 改进建议

- {{改进 1}}
- {{改进 2}}

### 质量指标

- 代码覆盖率: {{%}}
- 已发现和修复的缺陷: {{数}}
- 技术债: {{描述}}

### 团队反馈

{{团队成员的引用}}

### 后续行动

- [ ] {{行动 1}}
- [ ] {{行动 2}}
```

### 阶段 4：完成和存档

- [ ] 更新冲刺状态文件 - 标记史诗回顾为"已完成"
- [ ] 保存所有回顾备注
- [ ] 归档相关文档
- [ ] 通知利益相关者完成
- [ ] 安排下一个史诗的启动

## 输出文件

- **回顾备注** - 保存在 `retrospectives/epic-{{号}}-retrospective.md`
- **更新的冲刺状态** - 标记 `epic-{{号}}-retrospective: completed`

## 工作流利益

1. **庆祝** - 承认团队成就和努力
2. **学习** - 从经验中捕获可操作的见解
3. **连续改进** - 应用学习到未来的工作
4. **文档** - 创建可参考的项目历史
5. **团队建立** - 加强团队凝聚力和协作

## 回顾模式元素

### 庆祝元素
- 确认团队完成了艰巨的工作
- 表扬具体的贡献和成就
- 分享成功故事

### 反思元素
- 问开放式问题鼓励思考
- 积极聆听而不中断
- 记录学到的经验

### 改进元素
- 确定流程改进的具体领域
- 优先考虑改进（选择 3 个最重要的）
- 为每个改进分配所有者和截止日期

## 后续行动

1. **立即** - 分享回顾发现和关键学习
2. **本周** - 制定改进计划
3. **下个冲刺** - 实施已同意的改进
4. **存档** - 保存回顾记录供未来参考
