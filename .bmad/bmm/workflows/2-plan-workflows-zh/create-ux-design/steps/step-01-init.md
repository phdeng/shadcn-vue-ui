- 🛑 绝不生成没有用户输入的内容

- 📖 关键：始终在采取任何行动前阅读完整步骤文件 - 部分理解会导致不完整的决策
- 🔄 关键：加载下一步时（使用 'C'），确保完整阅读和理解整个文件后再进行
- ✅ 始终将此视为 UX 协调员和利益相关者之间的协作发现
- 📋 您是 UX 协调员，而不是内容生成器
- 💬 专注于初始化和设置 - 不要查看未来的步骤
- 🚪 检测现有工作流状态并正确处理继续

## 执行协议:

- 🎯 采取任何行动前显示您的分析
- 💾 初始化文档并更新前置信息
- 📖 设置前置信息 `stepsCompleted: [1]`，然后加载下一步
- 🚫 禁止在设置完成前加载下一步

## 上下文边界:

- 工作流 .md 中的变量在内存中可用
- 前一个上下文 = 输出文档中的内容 + 前置信息
- 不要假设来自其他步骤的知识
- 输入文档发现在此步骤中发生

## 您的任务：

通过检测继续状态并设置设计规范文档来初始化 UX 设计工作流。

## 初始化序列：

### 1. 检查现有工作流

首先，检查输出文档是否已经存在：

- 查找文件：`{output_folder}/ux-design-specification.md`
- 如果存在，读取完整文件（包括前置信息）
- 如果不存在，这是一个全新的工作流

### 2. 处理继续（如果文档存在）

如果文档存在且有 `stepsCompleted` 的前置信息：

- **立即停止**并加载 `./step-01b-continue.md`
- 不要进行任何初始化任务
- 让 step-01b 处理继续逻辑

### 3. 全新工作流设置（如果没有文档）

如果没有输出文档，继续初始化：

- 从模板创建新的 UX 设计规范文档
- 保存到 `{default_output_file}`
- 初始化前置信息，所有数组为空

### 4. 输入文档发现

从项目中发现上下文文档：

1. 查找产品需求文档（PRD）
2. 查找产品简介
3. 查找史诗和故事
4. 查找研究文档
5. 查找头脑风暴文档

对每个找到的文档：
- 读取完整内容
- 在前置信息的 `inputDocuments` 中记录
- 增加相应的 `documentCounts` 计数器

### 5. 展示发现成果

向用户展示：

"欢迎，{{user_name}}！我来帮您为 {{project_name}} 创建全面的 UX 设计规范。

**发现的上下文文档：**
- {{说出找到的文档数}}

**工作流准备就绪。我们准备好开始了吗？**"

### 6. 继续到步骤 2

准备加载下一步：

- 更新 `stepsCompleted: [1]` 和 `lastStep: 1`
- 加载 `./step-02-discovery.md`
