### 1. 从 PRD 加载用户旅程

使用 PRD 中已定义的用户旅程作为基础。

很好！由于我们有 PRD 可用，让我们建立在那里已经记录的用户旅程的基础上。

**PRD 中的现有用户旅程：**

我已经从您的 PRD 加载了这些用户旅程。

这些旅程告诉我们用户是**谁**以及为什么他们采取某些行动。现在我们需要设计那些旅程在细节中如何工作。

### 2. 设计每个旅程流

对于每个关键旅程，设计详细的流。

**对于 [旅程名称]：**

让我们设计用户完成 [旅程目标] 的流。

**流设计问题：**

- 用户如何开始这个旅程？(入口点)
- 他们在每个步骤需要什么信息？
- 他们需要做什么决定？
- 他们如何知道他们在成功？
- 这个旅程的成功看起来像什么？
- 他们可能在哪里感到困惑或卡住？
- 他们如何从错误中恢复？

### 3. 创建流程图

使用 Mermaid 图表可视化每个旅程。

我将为每个旅程创建详细的流程图，显示：

**[旅程名称]流：**

- 入口点和触发器
- 决策点和分支
- 成功和失败路径
- 错误恢复机制
- 信息的渐进式披露

### 4. 优化以实现效率和喜悦

完善用户体验的流。

**流优化：**

对于每个旅程，让我们确保我们：

- 最小化到价值的步骤 (快速让用户成功)
- 减少每个决策点的认知负荷
- 提供清晰的反馈和进度指标
- 创建喜悦或成就的时刻
- 优雅地处理边缘情况和错误恢复
