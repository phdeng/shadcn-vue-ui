# 步骤 1B：UX 设计工作流继续

## 强制执行规则（首先阅读）:

- 🛑 绝不生成没有用户输入的内容

- 📖 关键：在采取任何行动前始终阅读完整步骤文件
- 🔄 关键：加载下一步时（使用 'C'），确保阅读整个文件
- ✅ 始终将此视为 UX 协调员和利益相关者之间的协作发现
- 📋 您是 UX 协调员，而不是内容生成器
- 💬 专注于理解我们中断的位置并适当继续
- 🚪 从中断的确切位置恢复工作流

## 执行协议:

- 🎯 在采取行动前显示当前状态分析
- 💾 保持现有前置信息 `stepsCompleted` 值
- 📖 仅加载已在 `inputDocuments` 中跟踪的文档
- 🚫 禁止修改在前一步中完成的内容

## 上下文边界:

- 当前文档和前置信息已加载
- 前一个上下文 = 完整文档 + 现有前置信息
- 前置信息中列出的输入文档已处理
- 最后完成的步骤 = 前置信息中的 `lastStep` 值

## 您的任务：

从中断的位置恢复 UX 设计工作流，确保平滑继续。

## 继续序列：

### 1. 分析当前状态

查看前置信息以理解：

- `stepsCompleted`: 已完成哪些步骤
- `lastStep`: 最近完成的步骤编号
- `inputDocuments`: 已加载的上下文
- 所有其他前置信息变量

### 2. 加载所有输入文档

重新加载 `inputDocuments` 中列出的上下文文档

### 3. 欢迎用户回来

向用户展示进度摘要并询问是否继续

### 4. 确定下一步

基于 `lastStep` 值确定要加载的下一个步骤

### 5. 提交继续选项

询问用户是否准备好继续或进行任何调整
