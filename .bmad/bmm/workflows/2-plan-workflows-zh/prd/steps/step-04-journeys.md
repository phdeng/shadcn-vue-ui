# 步骤 4：用户旅程映射

**进度：步骤 4 / 11** - 下一步：领域需求

**步骤目标**: 映射与系统交互的所有用户类型，使用叙述故事式的旅程。

## 强制执行规则

- 🛑 永不生成没有用户输入的内容
- 📖 关键：始终阅读完整步骤文件
- ✅ 这是产品经理同行之间的协作发现
- 💬 关注映射所有与系统交互的用户类型
- 🎯 关键：没有旅程 = 没有功能需求 = 产品不存在

## 执行序列

### 1. 识别所有用户类型

与用户对话识别：

- 有多少不同类型的用户与此系统交互？
- 他们各自的目标是什么？
- 他们是否有不同的需求或约束？

### 2. 映射用户旅程

对于每种用户类型：

"让我们为[用户类型]映射旅程。

- 他们如何开始使用此系统？
- 他们尝试完成什么任务？
- 成功对他们意味着什么？
- 他们可能在哪里遇到问题？"

### 3. 记录旅程叙述

创建叙述故事式的旅程描述：

```markdown
## 用户旅程

### [用户类型 1]

**角色：** [描述]
**目标：** [他们想实现什么]
**旅程：** [逐步叙述他们如何使用系统]
**成功：** [什么表示成功]

### [用户类型 2]

[重复上述内容]
```

### 4. 验证覆盖范围

- 是否映射了所有用户类型？
- 是否有用户我们遗漏了？
- 旅程是否涵盖了所有关键用例？

### 5. 呈现内容和菜单

"我已为{{project_name}}映射了用户旅程。这些将指导我们的需求。

**选择一个选项：**
[A] 高级启发 - 让我们完善旅程映射
[P] 党派模式 - 引入不同的观点
[C] 继续 - 保存并移到领域需求（步骤 5 / 11）"

### 6. 处理菜单选择

- **如果 C**：追加内容，更新 `stepsCompleted: [1, 2, 3, 4]` 加载下一步
