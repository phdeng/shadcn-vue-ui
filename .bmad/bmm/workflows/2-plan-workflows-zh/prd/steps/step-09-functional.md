# 步骤 9：功能需求合成

**进度：步骤 9 / 11** - 下一步：非功能需求

**步骤目标**: 将所有发现综合成全面的功能需求。

## 强制执行规则

- 🛑 永不生成没有用户输入的内容
- 📖 关键：始终阅读完整步骤文件
- ✅ 这是产品经理同行之间的协作发现
- 💬 关注综合发现成功能性需求

## 执行序列

### 1. 检查现有发现

查看前面步骤中收集的所有内容：

- 用户旅程
- 成功标准
- MVP 范围
- 领域和项目类型需求

### 2. 为每个用户旅程合成功能需求

对于每个用户旅程：

- 用户需要什么功能来完成他们的任务？
- 系统需要什么功能来支持此旅程？

### 3. 记录功能需求

```markdown
## 功能需求

### [用户旅程 1] 的功能

- [功能 1] - [描述及其为何需要]
- [功能 2] - [描述及其为何需要]

### [用户旅程 2] 的功能

[重复上述内容]

### 跨功能需求

- [跨功能需求 1]
- [跨功能需求 2]
```

### 4. 验证功能覆盖范围

- 所有用户旅程都有功能吗？
- 是否有不支持任何旅程的功能？
- 所有 MVP 功能都已定义吗？

### 5. 呈现内容和菜单

"我为{{project_name}}合成了功能需求。

**选择一个选项：**
[A] 高级启发 - 让我们完善功能需求
[P] 党派模式 - 引入不同的观点
[C] 继续 - 保存并移到非功能需求（步骤 10 / 11）"

### 6. 处理菜单选择

- **如果 C**：追加内容，更新 `stepsCompleted: [1, 2, 3, 4, 5, 6, 7, 8, 9]` 加载下一步
