# 步骤 3：成功标准定义

**进度：步骤 3 / 11** - 下一步：用户旅程映射

**步骤目标**: 定义涵盖用户、业务和技术成功的全面成功标准。

## 强制执行规则

- 🛑 永不生成没有用户输入的内容
- 📖 关键：始终阅读完整步骤文件
- ✅ 始终将此视为产品经理同行之间的协作发现
- 💬 关注定义什么对此产品获胜

## 执行序列

### 1. 收集成功指标的想法

与用户对话确定：

- **用户成功指标：** 用户何时感到成功？
- **业务成功指标：** 企业如何衡量成功？
- **技术成功指标：** 系统应该有多快或可靠？
- **采用成功指标：** 有多少用户使用此产品？

### 2. 定义成功标准

基于对话：

```markdown
## 成功标准

### 用户成功

- [用户成功指标 1]
- [用户成功指标 2]
- [用户成功指标 3]

### 业务成功

- [业务成功指标 1]
- [业务成功指标 2]

### 技术成功

- [技术成功指标 1]
- [技术成功指标 2]
```

### 3. 呈现内容和菜单

"我已定义了{{project_name}}的成功标准。这些将指导我们的决定。

**选择一个选项：**
[A] 高级启发 - 让我们深化成功指标
[P] 党派模式 - 引入不同的观点
[C] 继续 - 保存并移到用户旅程映射（步骤 4 / 11）"

### 4. 处理菜单选择

- **如果 A**：执行高级启发，改进内容
- **如果 P**：执行党派模式，协作改进
- **如果 C**：追加内容，更新 `stepsCompleted: [1, 2, 3]` 加载下一步
