# Console 架构设计方案（2026-2029）

> 由 AI 架构评审团队产出，基于三维分析：现状审计 × 行业对标 × 趋势前瞻

## 一、推荐的模块划分

```
概览 Dashboard
────────────────────────────
模型 ▾（MLOps 全链路）
  ├── 模型目录        /models
  ├── 数据集          /datasets
  ├── 模型训练        /training
  ├── 模型评测        /evaluation
  └── 模型部署        /deployments
────────────────────────────
Agent ▾（一级分组）
  ├── Agent 管理      /agents
  ├── 工作流编排      /workflows      ← 新增
  └── Playground     /playground     ← 原 ChatAgent
────────────────────────────
知识库               /knowledge
工具与集成           /tools           ← 原插件市场
可观测性             /observability   ← 原推理监控
────────────────────────────
安全合规             /security        ← 新增
系统设置             /settings
```

## 二、与当前的差异

| 类型 | 改动 | 理由 |
|------|------|------|
| 改名 | 模型管理 → 模型目录 | 行业惯例 |
| 改名 | ChatAgent → Playground | 6/6 平台通用术语 |
| 改名 | 插件市场 → 工具与集成 | 无市场生态 + MCP 趋势 |
| 改名 | 推理监控 → 可观测性 | 覆盖追踪/成本/告警 |
| 升级 | Agent → 一级分组 | 3 方共识最强 |
| 新增 | 工作流编排 | 5/6 平台标配 |
| 新增 | 安全合规 | AI 法规 ★★★★★ |
| 移动 | Playground 归入 Agent 组 | Agent 调试入口 |

## 五、架构弹性设计

- 导航配置增加 `badge`/`permission`/`featureFlag` 字段
- NavGroup 支持多层嵌套
- 路由按模块拆分文件（`routes/agents.ts`）
- Store 按模块建立（`stores/training.ts`）
- 抽象通用 DataTable / FormBuilder 组件
