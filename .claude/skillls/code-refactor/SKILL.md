# Code Refactor Skill

扫描代码、识别重复、提取抽象、输出优化后的通用组件。

## 核心工作流程

```
1. 扫描分析 → 识别代码文件和结构
2. 模式检测 → 找出重复代码、相似逻辑
3. 抽象设计 → 设计通用接口/组件
4. 重构输出 → 生成优化后的代码
5. 验证检查 → 确保功能等价性
```

## 重复模式检测

### 检测目标

| 类型 | 示例 | 重构方向 |
|------|------|----------|
| **代码克隆** | 多处相同/相似代码块 | 提取函数/方法 |
| **相似逻辑** | 结构相同但参数不同 | 泛型/模板方法 |
| **重复模式** | 相同的异常处理/验证 | 装饰器/中间件 |
| **数据转换** | 多处相似的 map/filter | 通用转换器 |
| **配置散落** | 硬编码值散布各处 | 配置中心化 |

### 检测策略

```
1. 结构相似度：AST 级别的代码结构对比
2. 语义相似度：功能等价但写法不同的代码
3. 命名模式：相似命名暗示的重复逻辑
4. 调用链分析：重复的函数调用序列
```

## 重构策略（按语言）

### Python
- 重复函数 → 提取到 `utils/` 或创建基类
- 相似类 → 抽象基类 (ABC) + 模板方法
- 重复装饰逻辑 → 自定义装饰器
- 数据处理 → 通用 pipeline 函数

### JavaScript/TypeScript
- 重复组件逻辑 → Custom Hooks (React) 或 Composables (Vue)
- 相似 API 调用 → 通用 API 客户端
- 重复验证 → Schema 验证库 (zod/yup)
- 工具函数 → `utils/` 模块导出

### Java/Kotlin
- 重复代码 → 抽象类 + 模板方法模式
- 相似接口实现 → 泛型接口
- 重复配置 → Builder 模式或配置类

### Go
- 重复逻辑 → 提取到独立包
- 相似结构体方法 → 接口抽象
- 错误处理 → 统一错误处理函数

## 输出规范

### 重构报告格式

```markdown
## 📊 代码分析报告

### 发现的重复/优化点
1. [位置] - [问题描述] - [严重程度: 高/中/低]

### 重构建议
| 编号 | 当前问题 | 建议方案 | 预期收益 |
|------|----------|----------|----------|

### 提取的通用组件
- 组件名: [描述]
- 适用场景: [说明]
```

### 重构代码输出

每个重构后的组件/模块应包含：
1. 清晰的文件头注释说明用途
2. 类型定义（如适用）
3. 使用示例
4. 原代码位置的替换说明

## 质量检查清单

重构完成后验证：
- [ ] 功能等价：重构后行为与原代码一致
- [ ] 测试覆盖：关键路径有测试
- [ ] 命名清晰：抽象后的名称表意准确
- [ ] 文档完整：使用方式有说明
- [ ] 依赖合理：不引入循环依赖

## 使用提示

- 小范围重构：直接分析并输出优化代码
- 大型代码库：先输出分析报告，确认后再逐步重构
- 优先处理高频重复和高影响区域
