# 输出模板

本文件包含 Claude Code 在使用 vue-design-system-ops skill 时的默认输出模板。
根据任务规模选择对应模板。

---

## 模板 A：简版执行（S 级）

适用：单文件修改、样式微调、typo 修复、小 bug 修复

```markdown
## 变更摘要
[一句话说明做了什么]

## 修改文件
- `path/to/file.vue` — [修改说明]
- `path/to/file.ts` — [修改说明]

## 自检
- [x] 类型完整
- [x] 样式使用 design token
- [x] 暗色模式正常
- [x] 无 breaking change

## 下一步
[如有后续建议，简要列出]
```

---

## 模板 B：标准迭代（M 级）

适用：单组件新增/增强、composable 开发、文档补齐

```markdown
## 需求摘要
[需求描述，含背景和目标]

## 假设与待确认项
- [假设1]（置信度：高/中/低）— 如不成立：[影响]
- [假设2]...

## 影响范围
- **直接修改**：[文件列表]
- **间接影响**：[依赖此模块的组件/页面]
- **无影响确认**：[明确不受影响的边界]

## 方案设计

### 组件 API
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| ... | ... | ... | ... |

| Emit | Payload | Description |
|------|---------|-------------|
| ... | ... | ... |

| Slot | Scope | Description |
|------|-------|-------------|
| ... | ... | ... |

### Design Token 映射
[使用的 token 及其语义]

### 实现方案
[技术方案简述]

## 任务分解
1. [任务1] — 预估复杂度：[低/中/高]
2. [任务2] — 预估复杂度：[低/中/高]
3. ...

## 验收标准
- [ ] [可验证的条件1]
- [ ] [可验证的条件2]
- [ ] 暗色模式正常
- [ ] 键盘可完全操作
- [ ] TypeScript 无错误

## 主动扩展检查结果
### 必做
- [自动纳入的关联工作]

### 建议（待用户确认）
- [建议1]
- [建议2]

## 风险
- [风险1] — 缓解：[...]

## 可直接执行的下一步
[明确、具体的下一步操作]
```

## 模板 C：Brainstorm（探索性需求）

适用：用户不确定要什么、功能探索、方案比较

```markdown
## 目标理解
[对用户目标的理解和解读]

## 约束条件
- [技术约束]
- [设计系统约束]
- [时间/复杂度约束]

## 方案比较

| 维度 | 方案 A: [名称] | 方案 B: [名称] | 方案 C: [名称] |
|------|---------------|---------------|---------------|
| 核心思路 | | | |
| 复杂度 | ⭐⭐ | ⭐⭐⭐ | ⭐ |
| 一致性 | 高 | 中 | 高 |
| 可扩展性 | 中 | 高 | 低 |
| 可访问性 | 完备 | 部分 | 完备 |
| 迁移成本 | 低 | 高 | 无 |
| DX 体验 | 好 | 一般 | 好 |

### 方案 A: [名称]
[详细描述、优势、劣势]

### 方案 B: [名称]
[详细描述、优势、劣势]

### 方案 C: [名称]
[详细描述、优势、劣势]

## 推荐方案
**方案 [X]** — 理由：[为什么推荐]

## 推荐方案的初步实施路径
1. [步骤1]
2. [步骤2]

## 需要用户确认的问题
1. [问题1]
2. [问题2]

## 确认后的下一步
[确认后将转入标准迭代模式]
```

## 模板 D：深度架构（L 级复杂重构/系统设计）

```markdown
## 背景与动机
[为什么需要做这次重构/系统设计]

## 现状分析

[现有代码结构、依赖关系]

### 现有问题

2. [问题2] — 影响：[...]

## 目标架构
[期望达到的状态，可附架构图]

## Gap 分析
| 领域 | 现状 | 目标 | Gap |
|------|------|------|-----|
| ... | ... | ... | ... |

## 迁移路径

### 阶段 1: [名称] — [目标]
- **范围**：[涉及的文件/模块]
- **任务**：
  1. [任务1]
  2. [任务2]
- **验收**：[如何确认阶段完成]
- **风险**：[...]
- **回滚点**：[如何安全回退]
- **预估**：[复杂度/时间]

### 阶段 2: [名称] — [目标]
...

### 阶段 3: [名称] — [目标]
...

## Breaking Changes 清单
| 变更 | 影响范围 | 迁移方式 | 阶段 |
|------|---------|---------|------|
| ... | ... | ... | ... |

## 依赖关系图
[哪些任务可以并行，哪些有先后关系]

## 风险矩阵
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| ... | 高/中/低 | 高/中/低 | ... |

## 文档更新清单
- [ ] [需要更新的文档1]
- [ ] [需要更新的文档2]

## 测试策略
[回归测试范围和重点]

## 时间线建议
[分阶段的时间估算]

## 可直接执行的第一步
[最小、最安全的第一步操作]
```
