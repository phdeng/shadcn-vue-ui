# ADR（架构决策记录）模板

用于记录重要的技术决策，便于团队理解决策背景和后续追溯。

## 模板

```markdown
# ADR-[编号]: [标题]

**状态**: 提议中 | 已采纳 | 已废弃 | 已替代
**日期**: YYYY-MM-DD
**决策者**: [相关人员]

## 背景

[描述问题的背景和上下文，为什么需要做这个决策？]

## 决策驱动因素

- [因素1]
- [因素2]
- [因素3]

## 候选方案

### 方案A: [名称]

**描述**: [方案详细描述]

**优点**:
- [优点1]
- [优点2]

**缺点**:
- [缺点1]
- [缺点2]

### 方案B: [名称]

[同上格式]

## 决策

选择 **方案X**，因为：
- [理由1]
- [理由2]

## 影响

### 正面影响
- [影响1]

### 负面影响
- [影响1]

### 需要的后续行动
- [ ] [行动1]
- [ ] [行动2]

## 相关决策

- [ADR-XX: 相关决策标题]
```

## 使用指南

### 何时创建 ADR

- 技术选型（框架、数据库、中间件等）
- 架构模式选择（微服务 vs 单体、同步 vs 异步等）
- 重要的设计决策（API 风格、数据模型等）
- 影响多个团队或系统的决策

### 状态流转

```
提议中 → 已采纳 → 已废弃/已替代
```

### 最佳实践

1. **保持简洁**：每个 ADR 聚焦一个决策
2. **记录上下文**：包含决策时的背景信息
3. **版本控制**：将 ADR 纳入代码仓库
4. **定期回顾**：检查决策是否仍然有效
