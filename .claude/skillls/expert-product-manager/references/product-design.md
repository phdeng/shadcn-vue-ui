# 产品设计方法论

从0到1的产品设计系统方法。

## 产品设计流程

```
发现问题 → 定义问题 → 用户研究 → 方案设计 → 验证迭代
```

## 用户研究

### 用户画像框架

```markdown
## 基础信息
- 人口统计特征
- 行为特征
- 技术熟练度

## 场景分析
- 使用场景
- 触发时机
- 环境约束

## 需求分析
- 功能需求（要做什么）
- 情感需求（想获得什么感受）
- 社会需求（想展示什么形象）

## 痛点与期望
- 当前痛点（按严重程度排序）
- 理想状态
- 迁移障碍
```

### Jobs to be Done (JTBD)

用户"雇佣"产品完成特定任务：

```
当我处于 [情境] 时，
我想要 [动机/目标]，
以便于 [期望结果]。
```

**示例**：
```
当我在通勤路上时，
我想要快速获取今日重要新闻，
以便于利用碎片时间保持信息同步。
```

## 功能设计

### 功能价值评估矩阵

| 维度 | 权重 | 评估问题 |
|------|------|----------|
| 用户价值 | 40% | 解决多少用户的多大问题？|
| 商业价值 | 25% | 对收入/增长的贡献？|
| 战略价值 | 20% | 对长期竞争力的影响？|
| 实现成本 | 15% | 开发、维护、运营成本？|

### MVP定义原则

**包含**：
- 核心价值验证的最小功能集
- 完整的核心用户路径
- 基本的体验保障

**不包含**：
- 边缘场景处理
- 高级功能
- 完善的错误处理
- 极致的性能优化

### 功能规格模板

```markdown
## 功能名称
[清晰的功能命名]

## 用户故事
作为 [用户角色]，
我希望 [功能描述]，
以便 [价值/目的]。

## 验收标准
- [ ] 标准1
- [ ] 标准2
- [ ] 标准3

## 交互说明
[关键交互流程描述]

## 边界条件
- 异常情况处理
- 极端值处理
- 权限控制
```

## 用户体验设计

### 体验设计原则

1. **一致性** - 相似操作保持一致的交互模式
2. **可见性** - 系统状态清晰可见
3. **反馈性** - 操作有即时、明确的反馈
4. **容错性** - 预防错误，允许撤销
5. **简洁性** - 减少认知负荷，突出核心

### 用户旅程图模板

```
阶段: 认知 → 考虑 → 决策 → 使用 → 推荐

每阶段分析:
- 用户行为: 用户在做什么
- 接触点: 通过什么渠道
- 情绪曲线: 情绪高低
- 痛点: 遇到的问题
- 机会点: 可优化的地方
```

## AI产品特有设计要点

### AI能力边界设计

- **能力展示**：明确告知AI能做什么
- **预期管理**：设定合理的用户预期
- **失败处理**：优雅处理AI能力不足的场景

### 人机协作模式

| 模式 | 描述 | 适用场景 |
|------|------|----------|
| AI主导 | AI完成，人审核 | 标准化任务 |
| 人主导 | 人操作，AI辅助 | 创意性任务 |
| 协同 | 交替进行 | 复杂任务 |

### 信任建立机制

- 透明性：解释AI决策依据
- 可控性：允许用户修正AI输出
- 渐进性：从低风险任务建立信任
