# 产品评审框架

系统化的产品评审方法论，用于PRD评审、方案诊断、竞品分析。

## PRD评审清单

### 完整性检查

| 模块 | 检查项 | 状态 |
|------|--------|------|
| 背景 | 业务背景清晰 | |

| 目标 | 目标可量化 | |
| 目标 | 成功指标定义 | |

| 用户 | 用户场景描述 | |

| 方案 | 边界条件处理 | |
| 排期 | 里程碑计划 | |
| 风险 | 风险识别与对策 | |

### 质量评估维度

**需求质量**
- 需求是否真实存在？有何数据支撑？
- 需求的优先级判断依据是什么？
- 是否遗漏了关键需求？

**方案质量**
- 方案是否解决了核心问题？
- 有无更简单的解决方案？
- 技术可行性如何？

**体验质量**
- 用户路径是否顺畅？
- 关键操作的认知负荷如何？
- 异常场景处理是否完善？

**商业质量**
- ROI是否合理？
- 与业务目标的关联度？
- 对核心指标的预期影响？

## 评审问题库

### 战略层问题

- 这个功能与产品愿景的关系是什么？
- 为什么是现在做？时机是否合适？
- 竞品是否有类似功能？我们的差异化在哪？
- 不做这个功能的代价是什么？

### 范围层问题

- MVP范围是否足够小？
- 功能边界是否清晰？
- 是否存在功能蔓延的风险？
- 后续迭代计划是什么？

### 结构层问题

- 与现有功能的关系是什么？
- 信息架构是否合理？
- 是否增加了系统复杂度？
- 对其他模块的影响是什么？

### 框架层问题

- 核心用户路径是否清晰？
- 关键页面的信息层级是否合理？
- 导航逻辑是否清晰？

### 表现层问题

- 交互反馈是否及时？
- 视觉风格是否一致？
- 关键信息是否突出？

## 竞品分析框架

### 分析维度

```markdown
## 基础信息
- 产品定位
- 目标用户
- 核心功能

## 功能对比
| 功能点 | 竞品A | 竞品B | 我们 |
|--------|-------|-------|------|
| ... | ... | ... | ... |

## 体验对比
- 核心路径效率
- 上手难度
- 差异化体验

## 商业模式
- 盈利方式
- 定价策略
- 用户规模

## 启示
- 可借鉴的点
- 可超越的点
- 应避免的点
```

### 竞品选择原则

- **直接竞品**：同赛道、同定位
- **间接竞品**：不同方案解决相同问题
- **替代品**：用户的现有解决方案
- **跨界参考**：其他领域的优秀实践

## 问题诊断方法

### 问题定位流程

```
现象描述 → 数据验证 → 归因分析 → 根因确认 → 方案建议
```

### 常见问题模式

| 现象 | 可能原因 | 诊断方向 |
|------|----------|----------|
| 新用户流失 | 价值感知不足 | 首次体验、核心价值传达 |
| 活跃下降 | 使用动机减弱 | 使用场景、功能价值 |
| 功能使用率低 | 发现/使用门槛 | 入口、交互、引导 |
| 差评增加 | 预期不匹配 | 宣传、体验落差 |
