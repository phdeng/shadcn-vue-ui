# 战略规划框架

产品战略规划、路线图制定、GTM策略的系统方法。

## 战略定位

### 定位三角模型

```
        目标用户
           △
          / \
         /   \
        /     \
    价值主张 ─── 差异化优势
```

**定位声明模板**：
```
为 [目标用户]，
提供 [产品类型]，
解决 [核心问题]，
不同于 [竞品/替代方案]，
我们的 [独特价值]。
```

### 市场定位策略

| 策略 | 描述 | 适用场景 |
|------|------|----------|
| 领导者 | 定义品类标准 | 市场早期、资源充足 |
| 挑战者 | 差异化进攻 | 有明确领导者存在 |
| 跟随者 | 快速复制 | 资源有限、市场验证后 |
| 利基 | 聚焦细分市场 | 大市场难以进入 |

## 产品路线图

### 路线图类型

**时间驱动型**
- 按季度/年度规划
- 适合稳定发展阶段

**目标驱动型**
- 按里程碑组织
- 适合创业/创新项目

**主题驱动型**
- 按战略主题组织
- 适合平台型产品

### 路线图模板

```markdown
- 关键功能：
- 成功指标：

### Q2: [主题]
...
```

## 优先级排序

### ICE评分

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| Impact (影响) | 40% | 1-10，对目标的影响程度 |
| Confidence (信心) | 30% | 1-10，成功的把握程度 |
| Ease (容易度) | 30% | 1-10，实现的难易程度 |

**ICE = Impact × Confidence × Ease**

### RICE评分

```
RICE = (Reach × Impact × Confidence) / Effort

Reach: 影响用户数量
Impact: 对单用户的影响 (0.25/0.5/1/2/3)
Confidence: 估计的可信度 (50%/80%/100%)
Effort: 人月工作量
```

### 优先级矩阵

```
        高价值
           │
    快速胜利 │ 战略项目
   (优先做)  │ (规划做)
─────────────┼─────────────
    填充项目 │ 感谢不做
  (有空再做) │ (不要做)
           │
        低价值
   低成本 ──────── 高成本
```

## GTM策略

### 发布阶段

| 阶段 | 目标 | 关键动作 |
|------|------|----------|
| 内测 | 验证核心体验 | 小范围邀请、收集反馈 |
| 公测 | 验证增长模型 | 开放注册、监控指标 |
| 正式 | 规模化增长 | 全面推广、优化漏斗 |

### 增长飞轮设计

```
新用户获取 → 激活转化 → 留存活跃 → 推荐传播 → 新用户获取
     ↑                                        │
     └────────────────────────────────────────┘
```

### 北极星指标选择

好的北极星指标特征：
- 反映用户获得的核心价值
- 可被团队行动影响
- 领先于收入指标
- 简单、可理解

**示例**：
- Spotify: 听歌时长
- Airbnb: 预订夜数
- Slack: 日活跃用户发送消息数

## 风险管理

### 风险识别框架

| 风险类型 | 示例 | 应对策略 |
|----------|------|----------|
| 市场风险 | 需求不存在/变化 | 小步验证、持续调研 |
| 技术风险 | 实现困难/不稳定 | 技术预研、备选方案 |
| 竞争风险 | 巨头入场、价格战 | 差异化、建立壁垒 |
| 资源风险 | 人力/资金不足 | 优先级、外部合作 |
| 合规风险 | 政策变化、安全问题 | 合规预判、风控机制 |

### 风险评估矩阵

```
        高影响
           │
    关注监控 │ 重点管理
   (制定预案)│ (立即行动)
─────────────┼─────────────
    接受风险 │ 持续跟踪
   (定期回顾)│ (降低影响)
           │
        低影响
   低概率 ──────── 高概率
```

## 决策框架

### 重大决策清单

做出重大产品决策前，回答以下问题：

1. 这个决策与产品愿景一致吗？
2. 数据支持这个决策吗？
3. 用户会从中获益吗？
4. 我们有能力执行吗？
5. 最坏情况我们能承受吗？
6. 有更好的替代方案吗？
7. 现在是最佳时机吗？

```markdown
## 背景

|------|------|------|------|

| B | ... | ... | ... |

## 决策结论
[选择了什么，为什么]

## 下一步
```
